Spock

Testing

What?

No, really?

  • Created in 2008 by Peter Niederwieser (Gradle)
  • Joined by Luke Daley(Gradle)
  • Spock 1.0 released in 2015
  • Default Test framework in Grails
  • Used internally by Gradle, Groovy, etc.
  • Used by MongoDb, Tapestry, Netflix, JFrog

Hey, I really didn't know this, I found it on the Internet ;)

So how does it compare to JUnit

  • Compatible with _ALL_ JUnit tools (i.e. behaves like JUnit tests)
  • You can have a mixture of Spock and JUnit tests
  • They all run together
  • Supported by Maven, Gradle, Intellij, Eclipse, Sonar

So why should we use it?

  • Clearly marks phases (given, when, then, expect, ...)
  • Both sides of an assert are analyzed (better reporting than JUnit)
  • Built in Mocking
  • Parameterized tests (yes, JUnit has this, but it's less readable)
